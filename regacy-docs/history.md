# AI Skills Factory - 프로젝트 히스토리

## 2026-01-28

### 프로젝트 시작
- **목적**: Anthropic SKILLS를 보관하고 관리할 수 있는 게시판 형태의 웹 애플리케이션 개발
- **주요 기능**: SKILL CRUD, GitHub 연동, AI 추천, 검색/필터링, 즐겨찾기, 복사, 공유

### 요구사항 구체화 완료

- 기술 스택 결정:
  - Frontend: Next.js 15 + TypeScript + shadcn/ui + Tailwind CSS
  - Backend: Supabase (PostgreSQL + Auth)
  - AI: Google Gemini 2.0 Flash
  - External API: GitHub API
  - Deployment: Vercel

- `requirements.me`: 전체 요구사항 명세서

### 주요 결정 사항

#### 1. 데이터 저장 방식
- GitHub 파일을 직접 저장하지 않고 **GitHub URL만 저장**
- 이유: 저장 공간 절약, 원본 동기화 용이
- 고도화: MCP 형태로 파일 열람 기능 추가 예정

#### 2. AI 추천 시스템
- GitHub URL 입력 → GitHub API로 SKILL.md 파싱 → Gemini AI로 카테고리/태그 추천
- 사용자는 AI 추천을 수정 가능
- AI 실패 시 재시도 버튼 또는 수동 입력

#### 3. 카테고리 시스템
- 계층 구조: 최대 5depth
- 입력 형식: `개발>프론트엔드>React>Hooks>최적화`
- 저장 형식: `"['개발', '프론트엔드', 'React', 'Hooks', '최적화']"` (문자열)
- 사용자가 자유롭게 생성 가능, AI가 기존 카테고리 우선 추천

#### 4. UI/UX 설계
- **레이아웃**: 사이드바(카테고리 트리) + 탭(전체/내/공유받은) + 카드 그리드
- **검색**: 실시간 검색, 탭별 독립 검색
- **정렬**: 최신순, 오래된순, 이름순, 즐겨찾기순
- **카드**: SKILL 이름, 설명 요약, 카테고리, 즐겨찾기 버튼

#### 5. 공유 기능
- 이메일 주소로 SKILL 공유
- 받은 사람은 "공유받은 SKILLS" 탭에서 확인
- 공유받은 SKILL도 즐겨찾기 및 복사 가능
- Phase 1에서는 알림 기능 제외

#### 6. 복사 기능 (Fork 대신)
- 다른 사용자의 SKILL을 내 SKILL로 복사
- 원본 추적 없음 (단순 복사)
- 복사 후 완전히 독립적으로 관리
