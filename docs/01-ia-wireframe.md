# AI Ops Platform IA (MVP v1)

## 목표
- 1차 릴리스는 화면을 `Dashboard`와 `Skill` 두 개로 제한한다.
- 구조는 Skill 전용 서비스가 아닌, 향후 Agent/Provider/Policy 모듈 확장을 전제로 한다.

## 라우트 구조
- `/dashboard`
- `/skills`

## 글로벌 내비게이션
- 상단 탭: `Dashboard | Skill`
- 우측 공통 영역: 사용자 메뉴, 알림(실패 작업 개수), 빠른 실행(챗봇 열기)
- 탭 구조는 고정하고 이후 `Agents`, `Providers`, `Policies`를 같은 레벨로 추가 가능하게 설계

## 화면 1: Dashboard
### 목적
- 운영 상태를 빠르게 파악하고, 즉시 조치를 실행하는 지휘 화면

### 레이아웃
1. 상단 상태 카드 영역
- 총 Skill 수
- 연결된 Provider 수 / 전체 Provider 수
- 최근 24시간 설치 성공률
- 실패 작업 수

2. 중단 실행 현황 영역
- 최근 실행 작업 5~10개 테이블
- 컬럼: 실행 시각, Skill, 대상 Provider, 결과, 소요 시간, 실행 주체(사용자/챗봇)

3. 하단 빠른 조치 영역
- 실패 작업 다시 시도
- 전체 동기화 실행
- Skill 화면으로 이동

4. 우측(또는 하단) 챗봇 패널 진입
- 운영 명령 예시 버튼
- 예: "실패한 설치만 재시도", "어제 대비 실패 원인 요약"

## 화면 2: Skill
### 목적
- Skill 등록, 검색, 설치, 버전, 동기화를 실제로 관리하는 작업 화면

### 레이아웃
1. 상단 제어 영역
- 검색 (이름/태그/GitHub URL)
- 필터 (상태, Provider, 태그)
- 정렬 (최근 업데이트/이름/실패 우선)
- 기본 액션: `새 Skill 등록`, `일괄 설치`, `일괄 동기화`

2. 좌측 목록 영역
- Skill 리스트 (이름, 태그, 최신 버전, 상태 뱃지)
- 선택된 Skill 강조

3. 우측 상세 영역 (선택 Skill)
- 기본 정보: 이름, 설명, GitHub URL, 추적 브랜치/태그
- 버전 정보: 현재 고정 버전, 마지막 동기화 버전
- 설치 상태: Provider별 성공/실패/대기
- 실행 로그: 최근 N개 실행 이력 및 에러 메시지

4. 상세 액션 영역
- 단일 설치
- Provider 다중 선택 설치
- 실패 대상만 재시도
- 특정 버전 롤백

## 확장 대비 IA 원칙
- Dashboard는 모듈 통합 뷰, Skill은 모듈 상세 뷰로 역할 분리
- Provider 자체 관리 화면이 없어도, Skill 화면에서 Provider 상태를 다루되 데이터 모델은 분리
- 챗봇은 두 화면 어디서든 동일 Action 엔진을 호출하도록 설계
