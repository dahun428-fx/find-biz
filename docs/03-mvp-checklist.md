# MVP 확정 체크리스트

## A. 화면/UX
- [ ] 화면은 `Dashboard`, `Skill` 2개만 제공
- [ ] 상단 탭 `Dashboard | Skill` 고정
- [ ] Dashboard에서 Skill 운영 요약(카드 + 최근 실행) 확인 가능
- [ ] Skill 화면에서 등록/설치/재시도/롤백이 가능
- [ ] 챗봇 진입점이 양 화면에 존재

## B. 기능(필수)
- [ ] GitHub URL 기반 Skill 등록 (`SKILL.md` 파싱)
- [ ] Skill 목록/검색/필터/정렬
- [ ] 단일 설치
- [ ] 다중 Provider 일괄 설치
- [ ] 실패 대상 재시도
- [ ] 버전 롤백
- [ ] 설치 실행 이력/로그 조회

## C. 데이터/도메인(필수)
- [ ] Skill 엔터티 분리
- [ ] Provider 엔터티 분리
- [ ] Skill-Provider 설치 상태 매핑 테이블
- [ ] 설치 실행(Job) 이력 테이블
- [ ] 챗봇 실행 액션 감사 로그

## D. 챗봇(필수)
- [ ] 자연어 명령 5개 이상 지원
- [ ] 명령 실행 전 액션 파라미터 검증
- [ ] 실행 결과를 구조화된 요약으로 응답
- [ ] 실패 시 원인 + 다음 조치 제안

## E. 운영/안정성(필수)
- [ ] 비동기 Job 상태머신(`queued/running/success/failed/partial_failed`)
- [ ] 재시도 정책(최대 3회)
- [ ] 부분 실패 처리 및 UI 표시
- [ ] 기본 권한 체계(일반/관리자 또는 소유자/협업자)

## F. 확장성 가드레일
- [ ] Dashboard를 모듈 통합 뷰로 유지
- [ ] Skill 전용 로직과 공통 실행 엔진 분리
- [ ] Provider Adapter 인터페이스 고정
- [ ] 미래 모듈(`Agents`, `Policies`) 추가 시 라우팅/탭 충돌 없음

## MVP 제외(명시)
- [ ] 고급 승인 워크플로
- [ ] 카나리/점진 배포
- [ ] 비용 분석 리포트
- [ ] 고급 정책 엔진
